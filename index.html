JavaScript terms explained

in an unconventional way

IIFE

key word: instant

IIFE is a JavaScript function that runs as soon as it is defined (source)

(function () { 
    const name = "Ana";
})();

Like instant coffee where you just add water and your coffee is ready to drink, IIFE are functions that run immediately from the moment that you created them, there is no need to call them.

Like the instant coffee, you get one cup, after you drink it, that's it. Same with IIFE, as it's an anonymous function, you can't call it again after.

Closure

key word: access

A closure is a feature in JavaScript where an inner function has access to the outer (enclosing) function's variables — a scope chain (source)

For every closure we have three scopes:

  • Local Scope (Own scope)
  • Outer Functions Scope
  • Global Scope

Imagine the squirrel's house being the inner function; from there, the squirrel has access to the entire tree (which is like the outer function) so she can take the acorn (variable) if she wants to. She also have access to the rest of the forest(global scope).

Event bubbling

key word: propagate

When an event happens on an element, it first runs the handlers on it, then on its parent, then all the way up on other ancestors (source)

<form onclick = "alert('third')">FORM
    <div onclick = "alert('second')">DIV
        <button onclick = "alert('first')">Click me</button>
    </div>
</form>

Imagine a water drop touching the surface of the lake. This event (the drop touching the surface) is propagating, causeing ripples to appear. Same way, with bubbling, an event is handled first by the inner element (child) and then propagated to the outer elements (parents).

Hoisting

key word: memory

Conceptually, for example, a strict definition of hoisting suggests that variable and function declarations are physically moved to the top of your code, but this is not in fact what happens. Instead, the variable and function declarations are put into memory during the compile phase, but stay exactly where you typed them in your code (source)

//Declaring the function on top, then calling it
function myName(name) {
    console.log("My name is " + name)
}
myName("Ana");
                
//Call the function first, then declare it
myName("Ana");
function myName(name) {
    console.log("My name is " + name)
}

Let's say you have a bowl with fruits, you can first eat (or use) whatever fruits you have, and then name them (declare what you have gotten) or you can name (declare) first all the fruits you have and then eat (use) them - this being the most recommended way if you want to be sure you don't screw things up.

Prototype

key word: inheritance

Nearly all objects in JavaScript are instances of Object; a typical object inherits properties (including methods) from Object.prototype, although these properties may be overridden (source)

function Parent() { 
    this.surname = "Smith";
    this.nationality = "English";
    this.eyeColor = "blue";
}
            
function Child() {
    this.firstName = "Mary";
}
            
Child.prototype = new Parent();

Child.prototype.eyeColor
//returns "blue"
